Validating the diagnostic gap

In order to verify the validity of the “Diagnostic Gap” measure, also referred to as “Dx Gap”, provided by Finddx, we computed an alternative diagnostic gap using data from the World Health Organization (WHO).

Defining the diagnostic gap

The TB diagnostic gap is the “the proportion of the population with the condition who are undiagnosed” 1. In other words, we can define the diagnostic gap as the gap between the estimated number of people who fall ill with TB each year and the number of people who are officially reported as having been diagnosed. This means that the diagnostic gap represents people who have TB but are not officially diagnosed, due to limitations or challenges in the healthcare system.

Data Acquisition

The data required to calculate a new measure of diagnostic gap from the WHO can be found in two separate data sets. The estimated total number of TB cases expected each year can be found in the ‘e_inc_num’ variable in the ‘WHO TB burden estimates [>1Mb]’ dataset:

Code 
who_estimates <- read_csv(
  "https://extranet.who.int/tme/generateCSV.asp?ds=estimates"
)

who_estimates <- who_estimates |>
  select(
    country,
    year,
    tb_estimated_cases = e_inc_num
  )

who_estimates

The total number of actual reported cases by health systems can be found in the ‘c_newinc’ variable in the ‘Case notifications [>2Mb]’ dataset:

Code 
who_notifications <- read_csv(
  "https://extranet.who.int/tme/generateCSV.asp?ds=notifications"
)

who_notifications <- who_notifications |>
  select(
    country,
    year,
    tb_notified_cases = c_newinc
  )

who_notifications

Calculating the diagnostic gap

For each country and year in the WHO datasets, the diagnostic gap can be computed as follows:

  • diagnostic gap = (estimated cases - notified cases) / estimated cases
Code 
who_dx_gap <- who_estimates |>
  inner_join(who_notifications, by = c("year", "country")) |>
  mutate(
    who_dx_gap = (tb_estimated_cases - tb_notified_cases) / tb_estimated_cases
  ) |>
  select(country, year, who_dx_gap)

who_dx_gap

Examining Finddx’s diagnostic gap measure

Before we can use the newly calculated diagnostic gap measure from WHO to validate Finddx’s gap measure, we must first analyse the distribution of Finddx’s diagnostic gap measure.

Code 
finddx_fixed |>
  ggplot(aes(rate_dx_gap)) +
  geom_density(fill = "steelblue", alpha = .5) +
  theme_minimal() +
  labs(x = "Finddx's diagnostic gap")

An outlier, with a diagnsoptic gap rate of 1.25, appears to be causing the data to be positively skewed. Further inspection reveals this outlier comes from data for the Russian Federation:

Code 
finddx_fixed |>
  select(country, rate_dx_gap) |>
  arrange(desc(rate_dx_gap)) |>
  slice(1)

This value for the Russian Federation, with a diagnostic gap above 1, implies that there were a negative number of TB cases notified. It is reasoned that this is therefore a data entry or calculation error, and will be removed from the analyses.

Code 
finddx_fixed <- finddx_fixed |>
  filter(country != "Russian Federation")
Question for Finddx

Is the diagnostic gap value for the Russian Federation a mistake, or is there an alternative reason for why it may be above 1?

Validating Finddx’s diagnostic gap measure

Finddx’s diagnostic gap measure can now be validated against the measure we have just calculated from the WHO data. The Pearson’s correlation coefficient between the two measures of -0.986 indicates a very strong relationship, validating the measure used by Finddx:

Code 
who_dx_gap_2020 <- who_dx_gap |>
  filter(year == 2020) |>
  mutate(
    country = if_else(
      country == "Democratic Republic of the Congo",
      "DRC",
      country
    )
  )

dx_gaps_joined <- finddx_fixed |>
  select(country, finddx_dx_gap = rate_dx_gap) |>
  left_join(who_dx_gap_2020) |>
  select(
    country,
    `Finddx diagnostic gap` = finddx_dx_gap,
    `WHO diagnostic gap` = who_dx_gap
  )

dx_gaps_joined |>
  correlate() |>
  slice(1) |>
  select(-`Finddx diagnostic gap`)

This strong relationship can also be seen on the following scatter plot:

Code 
dx_gaps_joined |>
  ggplot(aes(`Finddx diagnostic gap`, `WHO diagnostic gap`)) +
  geom_point(colour = "steelblue", size = 5, alpha = .5) +
  theme_minimal()
Question for Finddx

Why is the correlation between the WHO calculated measure and Finddx’s measure negative, implying the inverse of the diagnostic gap has been calculated?

Exploring the determinants of the diagnostic gap

An exploxatory analysis on the ‘TB Policy’ Excel dashboard supplied by Finddx was conducted.

Data selection

To intiate the explatory analysis, it was decided that a cross-sectional analysis, looking at a single time point, would first be conducted. It was decided that data for 2019 would be used for two reasons:

  1. The data supplied by Finddx contains many missing values, especially for the labatory variables, for 2020.
  2. The global effects of the Covid-19 pandemic began to take effect in 2020 2 and the effects on the TB data presented in the supplied data set are unkown.

The supplied data only included a measure of diagnositc gap for 2020, so the value calculated from the 2019 WHO data is used from this point forward.

Code 
who_dx_gap_2019 <- who_dx_gap |>
  filter(year == 2019) |>
  mutate(
    country = if_else(
      country == "Democratic Republic of the Congo",
      "DRC",
      country
    )
  )

finddx_2019 <- finddx_ts |>
  filter(year == 2019) |>
  pivot_wider(names_from = indicator, values_from = value) |>
  left_join(finddx_fixed) |>
  select(-year, -rate_dx_gap)

data_2019 <- finddx_2019 |>
  left_join(who_dx_gap_2019)

Collectively, these data indicate that the number of labs performing culture smear liquid tests might be a significant determinant of a countires diagnostic gap, explaining up to 27.5% of the variance.
